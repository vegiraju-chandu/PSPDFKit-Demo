Overview

PSPDFLibrary implements a sqlite-based full-text-search engine. You can register documents to be indexed in the background and then search for keywords within that collection. There can be multiple libraries, although usually one is enough for the common use case.

Note: PSPDFLibrary is not available in PSPDFKit Viewer.

Warning: Processing PDFs takes quite some memory. We advise against running this in the background on an iPad 1, due to its hard memory limitations.

tokenizer

This property customizes what tokenizer should be used. Defaults to the ‘unicode61’ tokenizer. The UNICODE61 tokenizer allows searching inside text with diacritics. http://swwritings.com/post/2013-05-04-diacritics-and-fts Sadly, Apple doesn’t ship this tokenizer with their sqlite builds. PSPDFKit will detect custom versions of sqlite and use the ‘unicode61’ tokenizer whenever possible, falling back to the default (“simple”) tokenizer if not. For the default tokenizer PSPDFKit will utilize CFStringTransform() to strip combining marks and improve searchability. This is slower and modifies the content, compared to the unicode61 tokenizer. If your content only contains English language, it might not make a difference what tokenizer you are using. Read more about this at https://github.com/PSPDFKit/PSPDFKit-Demo/wiki/How-to-enable-the-unicode61-tokenizer

@property (nonatomic, copy) NSString *tokenizer

Discussion

Warning: Once the database is created, changing the tokenizer property won’t have any effect.

libraryWithPath:

Returns an library for this given path. If no instance for this path exists yet, this method will create and return one. All subsequent calls will return the same method. Hence there will only be one instance per path. This method will return nil for invalid paths.

documentUIDsMatchingString:options:completionHandler:

Query the database for a match of searchString. Only direct matches, begins-with and ends-with matches are supported. Returns a dictionary of UID->NSIndexSet of page numbers.

- (void)documentUIDsMatchingString:(NSString *)searchString options:(NSDictionary *)options completionHandler:(void ( ^ ) ( NSString *searchString , NSDictionary *resultSet ))completionHandler

Discussion

Note: Ends-with matches are only possible if saveReversedPageText has been YES while the document was indexed.
